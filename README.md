# DoV1_extract

**Proof Of Concept & Work In Progress**

`DoV1_extract` is a command-line utility written in C++ for extracting Dolby Vision RPU (Reference Processing Unit) metadata from AV1 video streams. It parses AV1 bitstreams enclosed in an IVF container, locates the ITU-T T.35 OBU metadata payload containing Dolby Vision data (EMDF wrapper), and extracts it into a standard HEVC UNSPEC62 NAL unit binary format, completely compatible with tools like `dovi_tool`.

## Features
- Parses AV1 IVF containers.
- Identifies and unpacks Dolby Vision EMDF (Extensible Metadata Format) payloads from AV1 OBU Metadata (Type 5, ITU-T T.35).
- Restores proper HEVC NAL unit structure (`00 00 00 01 7C 01`).
- Injects Emulation Prevention Bytes (`0x03`) to ensure valid RPU payload bitstreams.
- Reorders Extracted RPUs based on presentation timestamps (PTS) to ensure correct presentation order.

## Usage

```bash
DoV1_extract -i {input_av1_file.ivf} -o {output_rpu.bin} [-v]
```

### Arguments
* `-i`: Path to the input AV1 video file (currently requires an **IVF** container).
* `-o`: Path to the output binary file where the extracted Dolby Vision RPU payload will be saved.
* `-v`: (Optional) Verbose mode. Prints extraction details frame by frame.
* `-h`: Shows the help message.

## How it works
1. **Container Parsing**: The tool reads the IVF container header to identify frames and their timestamps.
2. **OBU Parsing**: It iterates through the AV1 Open Bitstream Units (OBU) inside each frame to find `OBU_METADATA` (Type 5).
3. **T.35 & EMDF Unpacking**: It specifically targets ITU-T T.35 metadata belonging to Dolby Laboratories (Provider Code `0x003B`), performs bit-level parsing of the EMDF header, and retrieves the raw RPU payload.
4. **Formatting & Sorting**: Extracts the payload, prepends the `0x19` prefix, adds HEVC Start Codes and NAL headers (`00 00 00 01 7C 01`), inserts necessary emulation prevention bytes (`0x03`), and finally sorts all RPUs according to their IVF timestamps before writing to the output file.
